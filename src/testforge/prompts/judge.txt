You are a strict test-quality reviewer. You are given a Python function, a test that passes on it, and a list of mutated versions of the function that the test FAILED to detect (i.e., the test still passed even though the code is broken).

YOUR JOB
1. For each surviving mutation, explain in one sentence why the test missed it.
2. Propose a stronger replacement test that would kill all of them.

OUTPUT FORMAT (JSON, no markdown fences)
{
  "explanations": ["mutation 1 survived because ...", "mutation 2 survived because ..."],
  "improved_test": "from target import ...\n\ndef test_...():\n    assert ..."
}
